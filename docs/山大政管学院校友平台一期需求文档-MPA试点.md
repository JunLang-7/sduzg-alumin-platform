# 山大政管学院校友平台一期需求文档（MPA 试点）

版本：V0.2  
日期：2026-04-27  
阶段范围：第一阶段，MPA 校友数据平台试点  
后续方向：在 MPA 试点稳定后，扩展为政治学与公共管理学院全院校友平台

## 1. 文档目的

本文档根据当前沟通确认的第一阶段需求，重新整理山大政管学院校友平台一期建设范围、用户角色、权限边界、数据字段和核心功能，为后续设计、开发、测试和验收提供依据。

第一阶段不追求完整学院级校友生态，重点解决三类核心问题：

1. 管理员能够查看数据大屏，管理 MPA 校友基础信息。
2. 普通校友能够登录后查看所有 MPA 校友信息，并维护自己的个人信息。
3. 超级管理员能够管理管理员账号，保证平台权限可控。

## 2. 项目定位

一期平台定位为“山大政管 MPA 校友数据管理与查询平台”。

平台先以 MPA 校友为试点，沉淀账号体系、角色权限、校友数据字段、后台管理、数据大屏和校友端查询能力。待数据规范、权限规则和使用流程稳定后，再扩展到政管学院其他专业、项目和校友群体。

## 3. 一期建设范围

### 3.1 一期纳入范围

| 模块 | 说明 |
| --- | --- |
| 登录与权限 | 支持游客、校友、管理员、超级管理员四类角色 |
| 校友信息管理 | 管理员维护 MPA 校友基础资料 |
| 校友信息查询 | 登录校友查看所有 MPA 校友资料 |
| 个人信息维护 | 校友维护自己的资料 |
| 数据大屏 | 管理员查看 MPA 校友数据统计与分析 |
| 管理员管理 | 超级管理员创建、删除管理员账号 |
| 基础安全 | 登录校验、权限拦截、操作记录、数据备份 |

### 3.2 一期暂不纳入范围

以下能力不作为第一阶段验收目标，可作为后续扩展：

1. 全院校友数据覆盖。
2. 校友活动报名、签到和活动回顾。
3. 新闻、通知、校友故事等内容管理系统。
4. AI 校友卡、AI 找校友等智能化扩展能力。
5. 微信登录、统一身份认证、短信通知、邮件通知。
6. 校友组织、地区分会、行业分会管理。
7. 复杂审批流、工作流和多级数据审核。

## 4. 用户角色

一期角色分为四类：游客、校友、管理员、超级管理员。

| 角色 | 定义 | 核心权限 |
| --- | --- | --- |
| 游客 | 未登录用户 | 只能访问登录页及允许公开展示的基础页面，无法访问校友服务 |
| 校友 | 已登录的 MPA 校友 | 查看所有校友信息，维护自己的个人信息 |
| 管理员 | 平台运营管理人员 | 查看数据大屏，管理校友信息，查看所有校友资料 |
| 超级管理员 | 最高权限管理员 | 具备管理员全部权限，额外可创建和删除管理员 |

### 4.1 游客

游客为未登录默认身份。

需求：

1. 游客可访问登录页。
2. 游客可访问平台首页或说明页。
3. 游客无法访问校友服务功能。
4. 游客无法查看校友列表、校友详情、数据大屏、管理后台。
5. 游客访问受限页面时，应跳转登录页或显示无权限提示。

### 4.2 校友

校友为登录后的普通 MPA 校友用户。

需求：

1. 校友可登录平台。
2. 校友可查看所有 MPA 校友基础信息。
3. 校友可搜索、筛选、查看校友详情。
4. 校友可查看自己的个人信息。
5. 校友可修改自己的个人信息。
6. 校友不可新增、删除或修改其他校友的信息。
7. 校友不可访问管理员后台和数据大屏管理页。

说明：

当前需求为“普通校友查看所有校友信息”。从个人信息保护角度，对游客完全隐藏校友数据；登录校友可查看校友资料字段，但系统需保留后续脱敏扩展能力。

### 4.3 管理员

管理员用于日常维护 MPA 校友数据。

需求：

1. 管理员可登录管理后台。
2. 管理员可查看数据大屏。
3. 管理员可查看所有校友信息。
4. 管理员可新增、编辑、删除校友信息。
5. 管理员可搜索、筛选、分页查看校友列表。
6. 管理员可维护校友账号与校友档案的关联关系。
7. 管理员不可创建、删除管理员账号。
8. 管理员不可删除超级管理员账号。

### 4.4 超级管理员

超级管理员用于平台最高权限管理。

需求：

1. 超级管理员拥有管理员全部权限。
2. 超级管理员可创建管理员账号。
3. 超级管理员可删除管理员账号。
4. 超级管理员可查看管理员列表。
5. 超级管理员不可删除自己的账号。
6. 系统至少保留一个超级管理员账号。

## 6. 校友数据字段

一期以 MPA 校友字段为准。

| 字段 | 字段说明 | 是否必填 | 备注 |
| --- | --- | --- | --- |
| 姓名 | 校友真实姓名 | 必填 | 核心识别字段 |
| 年级 | 入学年级或在读年级 | 必填 | 如 2020 级 |
| 班级 | 所属班级 | 建议必填 | 如 MPA 2020 级周末班 |
| 届数 | 毕业届别 | 建议必填 | 如 2023 届；需与“年级”区分 |
| 辅导员 | 班级辅导员 | 选填 | 可用于班级管理 |
| 导师 | 学业导师或论文导师 | 选填 | 当前系统已有导师字段 |
| 专业 | 专业方向 | 建议必填 | 一期默认 MPA |
| 培养方式 | 全日制、非全日制等 | 必填 | 需根据学院标准枚举 |
| 行业 | 根据工作单位自动划分 | 必填 | 限制选项 |
| 工作单位 | 当前工作单位 | 选填 | 当前系统已有单位字段 |
| 职务 | 当前职务 | 选填 | 新增字段 |
| 通讯地址 | 联系地址 | 选填 | 注意隐私保护 |
| 性别 | 男、女、其他/未填 | 选填 | 当前系统已有字段 |
| 联系方式（手机） | 手机号码 | 建议必填 | 需考虑展示权限和脱敏 |

### 6.1 字段口径说明

1. “年级”建议定义为入学年级，例如 2020 级。
2. “届数”建议定义为毕业届别，例如 2023 届。
3. “培养方式”建议使用固定枚举，避免自由填写导致统计失真。
4. 手机号码应做格式校验，避免空格、短号、固定电话混入同一字段。

### 6.2 建议补充的系统字段

为满足管理和审计需求，系统应额外保存以下字段：

| 字段 | 说明 |
| --- | --- |
| 校友 ID | 系统唯一编号 |
| 账号 ID | 关联登录账号 |
| 创建时间 | 数据创建时间 |
| 更新时间 | 最近更新时间 |
| 创建人 | 管理员或系统导入 |
| 更新人 | 最近修改人 |
| 数据状态 | 正常、停用、删除 |
| 备注 | 管理员内部备注，可选 |

## 7. 功能需求

### 7.1 登录与访问控制

| 编号 | 需求 |
| --- | --- |
| FR-001 | 用户通过账号和密码登录。 |
| FR-002 | 系统根据登录账号识别角色：校友、管理员、超级管理员。 |
| FR-003 | 用户退出后清除登录状态。 |
| FR-004 | 未登录用户访问受限页面时跳转登录页。 |
| FR-005 | 无权限用户访问后台、大屏等页面时显示无权限提示。 |
| FR-006 | 用户可修改自己的密码。 |
| FR-007 | 登录失败时提示账号或密码错误，不暴露具体账户状态。 |

### 7.2 校友端：查看所有校友信息

| 编号 | 需求 |
| --- | --- |
| FR-101 | 校友登录后可进入校友信息列表。 |
| FR-102 | 校友可查看所有 MPA 校友信息。 |
| FR-103 | 校友列表支持分页展示。 |
| FR-104 | 校友列表支持按姓名、年级、班级、届数、导师、专业、培养方式、工作单位、职务等条件搜索。 |
| FR-105 | 校友可进入详情页查看单个校友完整资料。 |
| FR-106 | 校友不可编辑他人资料。 |
| FR-107 | 校友不可删除任何校友资料。 |

列表建议展示字段：

1. 姓名。
2. 年级。
3. 班级。
4. 届数。
5. 专业。
6. 培养方式。
7. 工作单位。
8. 职务。
9. 手机号。

详情页建议展示全部校友字段。

### 7.3 校友端：管理自己的信息

| 编号 | 需求 |
| --- | --- |
| FR-201 | 校友可查看自己的完整资料。 |
| FR-202 | 校友可编辑自己的资料字段。 |
| FR-203 | 校友保存资料后，系统更新本人档案。 |
| FR-204 | 系统记录修改时间和修改人。 |
| FR-205 | 手机号格式不正确时不允许保存。 |
| FR-206 | 姓名、年级等核心字段是否允许校友自行修改，需要最终确认。 |

一期建议：

1. 校友可直接修改联系方式、工作单位、职务、通讯地址。
2. 姓名、年级、班级、届数、专业、培养方式等基础学籍字段建议默认只读，由管理员维护。

### 7.4 管理端：校友信息管理

| 编号 | 需求 |
| --- | --- |
| FR-301 | 管理员可查看校友列表。 |
| FR-302 | 管理员可按姓名、手机号、年级、班级、届数、导师、专业、培养方式、工作单位等检索。 |

| FR-304 | 管理员可编辑校友信息。 |
| FR-305 | 管理员可删除校友信息。 |

## 11. 非功能需求

| 类别 | 需求 |
| --- | --- |
| 权限安全 | 所有受限页面和接口必须做服务端权限校验 |
| 密码安全 | 密码必须哈希存储，不能明文存储或输出日志 |
| 数据安全 | 游客不能通过接口直接获取校友数据 |
| 操作审计 | 管理员新增、修改、删除校友和管理员账号需记录日志 |
| 性能 | 校友列表分页查询应在 2 秒内返回 |
| 可维护性 | 年级、班级、专业、培养方式等字段应支持后续扩展 |
| 备份 | 校友数据需定期备份，删除操作建议软删除 |
| 兼容性 | 支持主流桌面浏览器，校友端应兼容移动端访问 |

## 12. 验收标准

一期完成时，至少满足以下标准：

1. 游客无法访问校友列表、校友详情、管理后台和数据大屏。
2. 校友登录后可查看所有 MPA 校友列表。
3. 校友可搜索校友信息。
4. 校友可查看校友详情。
5. 校友可维护自己的资料。
6. 校友无法修改他人资料。
7. 管理员可进入后台。
8. 管理员可新增、编辑、删除校友信息。
9. 管理员可查看数据大屏。
10. 超级管理员可创建管理员账号。
11. 超级管理员可删除管理员账号。
12. 管理员无法创建或删除管理员账号。
13. 系统至少保留一个超级管理员。
14. 关键操作有日志记录。
15. 密码不明文存储。
