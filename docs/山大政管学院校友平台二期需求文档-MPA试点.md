# 山大政管学院校友平台二期需求文档（MPA 试点）

版本：V0.2
日期：2026-06-07
阶段范围：第二阶段，在一期基础上扩展数据管理、文件归档与性能优化能力
前置依赖：一期（v0.1.0）已完成账号体系、角色权限、校友 CRUD、数据大屏和管理员管理

## 1. 文档目的

本文档覆盖一期发布（v0.1.0）后的新增需求与技术改造。二期不改变一期已有的角色模型和权限边界，而是在此基础上补齐批量操作、文件管理、敏感字段保护和性能优化四类能力。

二期核心目标：

1. 管理员能够批量导入和导出校友数据，提升数据维护效率。
2. 管理员能够为校友上传、下载学位档案和学籍档案等文件。
3. 普通校友查看他人资料时，手机号、职务、通讯地址等敏感字段受控隐藏。
4. 数据大屏交互与视觉升级，增强地域和行业维度分析。
5. 引入 Redis 缓存优化高频查询，降低数据库压力。

| FR-803 | 支持的附件格式为 PDF、Word (.doc/.docx)、JPG、PNG，单文件限制 50MB。 |

### 3.6 Redis 缓存优化

| 编号 | 需求 |
| --- | --- |
| FR-1101 | 校友总数（active 状态）通过 Redis 主动缓存，无筛选条件时列表 COUNT 查询直接读缓存。 |
| FR-1102 | 新增、删除、导入校友时通过 INCRBY 原子更新 Redis 计数器，无需重新 COUNT。 |
| FR-1103 | 导出数据查询结果通过版本号机制缓存，同一查询条件在数据未变更时直接返回缓存结果。 |
| FR-1104 | 数据变更时递增版本号，所有旧缓存自动失效（新查询生成新 key），旧 key 通过 TTL 自动清理。 |
| FR-1105 | 导出缓存 TTL 为 2 小时，适应数据大屏读多写少的场景。 |

## 6. 基础设施变更

### 6.1 MinIO 对象存储

- MinIO 作为 Docker 服务与 MySQL、Redis 统一编排
- 通过环境变量配置：`STORAGE_ENDPOINT`、`STORAGE_ACCESS_KEY`、`STORAGE_SECRET_KEY`、`STORAGE_BUCKET`
- 文件使用预签名 URL 下载，不暴露对象存储访问凭证
- 存储未启用时文件路由不注册，服务降级运行

### 6.2 Redis 缓存

- 新增 `alumni:count` key：缓存活跃校友总数，无 TTL（数据库状态为准，由 INCRBY/Set 维护）
- 新增 `alumni:export:version` key：版本号计数器，INCR 递增
- 新增 `alumni:export:{version}:{md5(query)}` key：导出查询结果缓存，TTL 2 小时

### 6.3 数据库变更

- 新增 `alumni_files` 表
- `alumni_profiles` 无结构变更
- 新增测试种子数据脚本 `002_seed_test_user.sql`

## 7. 安全加固

| 类别 | 措施 |
| --- | --- |
| 公式注入防护 | 导出时转义以 `=` `+` `-` `@` 开头的单元格值 |
| 路径穿越防护 | 文件 object key 过滤 `/` `\` `..` 字符 |
| 文件校验 | 通过魔数 `PK\x03\x04` 判断 xlsx 格式，限制 20MB/5000 行 |
| 文件类型白名单 | 仅允许 PDF/Word/JPG/PNG 上传，服务端校验 file_type 为 degree_archive 或 academic_record |
| 敏感字段保护 | fail-closed 策略，无法确认角色时默认屏蔽 |
| Content-Type | 上传请求不手动设置 Content-Type，避免 boundary 丢失 |

## 9. 验收标准

二期完成时，在满足一期全部验收标准的基础上，至少额外满足：

1. 管理员可通过 Excel 批量导入校友数据，含必填校验和四字段去重。
2. 导入完成后前端弹窗展示成功/失败/重复明细。
3. 管理员可导出筛选后的校友数据为 xlsx 或 csv 格式。
4. CSV 文件在 Excel 中打开不出现乱码。
5. 导出单元格不会触发电子表格公式注入。
6. 管理员可为校友上传、下载、删除学位档案和学籍档案文件。
7. 同一类型文件上传新版本时旧文件自动软删除。
8. 删除校友时其关联文件全部级联软删除。
9. 普通校友查看他人详情时，手机号、职务、通讯地址显示 `***`。
10. 管理员查看任意校友详情时，所有字段正常显示。
11. 无筛选条件时校友列表 COUNT 查询走 Redis 缓存。
12. 数据变更后导出缓存自动失效，新查询获得最新数据。
13. 文件上传 > 50MB 或非支持格式时服务端拒绝并返回明确错误提示。
14. 导入文件格式不符（非 xlsx）时服务端返回明确错误提示。
15. 操作日志包含文件上传和删除的审计记录。
